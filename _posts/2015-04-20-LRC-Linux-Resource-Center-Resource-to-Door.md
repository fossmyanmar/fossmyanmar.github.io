**LRC** ဆိုတာကို ဘယ်သူတွေ မှတ်မိလဲ။
စကားမစပ် LRC – Local Resources Centre (Myanmar) လဲ LRC လို့ခေါ်တယ်။ အဲတာကို ပြောတာမဟုတ်ပါဘူး။

အရင်တုန်းက ကျွန်တော်တို့ Linux Resource Center ဆိုပြီး လုပ်ခဲ့ကြတာပါ။
မိမိနှစ်သက်ရာ OS ကို National CD မှာတောင် ဝယ်မရတဲ့ အခြေအနေမှာ မျှဝေဖို့ ကြိုးစားခဲ့ကြတာပေါ့။

အခုခေတ်မှာတော့ Ubuntu ခွေ Fedora ခွေ CentOS ခွေ RedHat ခွေတွေကို လွယ်လွယ်ကူကူ ဝယ်လို့ရနေပါပြီ။ ဒါပေမယ့် Update မဖြစ်တာ Architect မစုံတာ သူတို့ မသိပါဘူး။ version အစုံလည်း မရှိပါဘူး။ Video Tutorial တွေ Guide ရေးပေးတဲ့ Website တွေ Wiki တွေဆိုတာလည်း အင်တာနက်ရမှ အသုံးပြုနိုင်တာပါ။ ဒီအတွက် LRC ရှိမယ်ဆိုရင် ဒါတွေကို Internet Traffic မရှိပဲ သုံးနိုင်ပါလိမ့်မယ်။

အဲအတွက်
LRC လေးတွေ ပြန်လုပ်ရင် ကောင်းမလား
LRC တွေလုပ်မယ်ဆိုရင်

- ၁) Storage အတွက် External HDD တော့ အနည်းဆုံး ကုန်ကျပါမယ်။ Internet ဆိုင်လို ကော်ဖီဆိုင်လိုနေရာတွေက ဝိုင်ဖိုင်တွေဆိုရင် ပါဝင်ဖွင့်ထားတဲ့အချိန်ရပါမယ်။
- ၂) File Server လိုထားမယ်ဆိုရင် Pi ထပ်လိုပါမယ်။
- ၃) ဆိုင်တွေ နေရာတွေက လက်ခံသလား တာဝန်ယူသလား ပူးပေါင်းပါဝင် ကူညီမလား ဆိုတာရှိပါတယ်။


သို့မဟုတ်
Door 2 Door Delivery လုပ်မလား စဉ်းစားရပါမယ်။
အိမ်တိုင်ရာရောက် ပို့ပေးပြီး တခွေကို ၁၀၀၀ ထက်မပိုတဲ့နှုန်းနဲ့ အမြန်ဆုံးပို့ပေးနိုင်ဖို့ ကြိုးစားကြည့်ကြပါမယ်။

LRC ရဲ့ အားသာချက်က LRC ရှိတဲ့ နေရာတွေ ကြေငြာထားမှာဖြစ်လို့ ကြုံတဲ့နေရာ ဝင်ပြီး ယူသွားယုံပါပဲ။ ကုန်ကျစားရိတ်ကို LRC နဲ့ ဆိုင်တွေဘက်က ကျခံရပါလိမ့်မယ်။

D2D ရဲ့ အားသာချက်က စားရိတ်အများကြီး မကုန်ပဲ အိမ်တိုင်ရာရောက် ပို့ပေးပါလိမ့်မယ်။ ကုန်ကျစားရိတ်ကို အသုံးပြုသူက ကျခံရပါလိမ့်မယ်။ အားနည်းချက်ကတော့ နယ်ကလာတဲ့လူဆိုရင် LRC က အဆင်ပြေတဲ့အချိန်လေး လမ်းကြုံဝင်ယူသွားရုံပါပဲ။ D2D ဆိုရင် လက်ခံမယ့် အိမ်တွေ ငွေချေပေးမယ့် လူတွေ လိုအပ်နေပါလိမ့်မယ်။

တကယ်လို့ တခုခုက ပိုကောင်းတဲ့ အကြောင်းပြချက်ရှိရင် ဒီဟာတွေထက် ကောင်းတဲ့ အကြံရှိရင် ပေးစေချင်ပါတယ်။ LRC လုပ်ဖို့ကို စိတ်ဝင်စားတဲ့လူတွေရှိတယ်ဆိုလည်း ဘယ်အပိုင်းက ဘယ်လိုကူညီပေးနိုင်တယ်ဆိုတာလေးပါ ရေးခဲ့ပေးနိုင်ပါတယ်။
